# 3.3.2 레이블, 팝업/Alert/Push

## 레이블 (버튼, 폼 필드 등)
- 행동을 촉구하는 동사 또는 명백한 명사 위주로 짧게 명시합니다. (예: "저장", "수정", "취소")
- 사용자가 버튼을 클릭했을 때 일어날 결과를 쉽게 예측할 수 있는 단어를 사용합니다.
- 입력 필드의 레이블은 해당 필드가 요구하는 정보가 무엇인지 직관적으로 보여주어야 합니다.

## 팝업 / Alert / Push
- **제목**: 사용자에게 발생한 중요한 상황이나 필요한 행동을 1~2줄 이내로 짧게 요약합니다.
- **본문**: 구체적인 이유나 결과, 향후 미칠 영향을 설명하고 긍정적인 대안 또는 행동 방향을 제시합니다.
- **버튼**: 단순히 "확인", "예/아니오" 대신, 본문을 읽지 않아도 버튼명 만으로 행동의 결과를 알 수 있도록 구성합니다. (예: "삭제하기", "유지하기")
