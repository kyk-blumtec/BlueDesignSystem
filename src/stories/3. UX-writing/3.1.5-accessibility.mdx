# 접근성

### 사용자 관점
<ul className="guide-list">
  <li>남녀노소, 디지털 취약자, 다양한 문화적 배경 사용자 등 누구나 이해할 수 있도록 쉽고 직관적인 문장을 사용합니다.</li>
  <li>약어, 전문 용어, 모호한 표현은 최소화하고, 화면 리더기·인지적 이해를 고려한 문장 구조로 작성합니다.</li>
  <li>서비스를 만드는 내부 관점이나 기술 용어가 아닌, 사용자가 이해하는 언어와 상황 중심으로 표현합니다.</li>
  <li>기능 중심 설명 대신, 사용자가 ‘무엇을 얻고 해결할 수 있는지’를 알려주는 방식으로 작성합니다.</li>
  <li>아이콘만으로 의미를 전달하지 않습니다.</li>
</ul>

### 행동 중심
<ul className="guide-list">
  <li>문장이 사용자의 ‘행동’을 즉시 유도할 수 있도록 명령형·행동형 문구를 사용합니다.</li>
  <li>버튼·알림·설명 문구 모두 ‘사용자가 지금 무엇을 할 수 있는지’를 명확히 알려야 합니다.</li>
</ul>
